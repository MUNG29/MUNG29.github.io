HttpServlet抽象类

GenericServlet类实现了javax.servlet.Servlet接口，为程序开发提供了方便，但是实际开发中更多的还是使用servlet处理http请求并作出响应，所以继承GenericServlet类仍不是很方便，而javax.servlet.http.HttpServlet类对GenericServlet类进行了继承和扩展，它也是一个抽象类，提供了doGet()、doPost()等方法

常用方法

1.获得信息

getRequestURL方法返回客户端发出请求时的完整URL。

getRequestURI方法返回主机名到请求参数之间部分。

getQueryString 方法返回请求行中的参数部分，如Myservlet?name=x，返回name=x。

getRemoteAddr方法返回发出请求的客户机的IP地址

getMethod得到客户机请求方式

getServerPath获取请求的文件的路径，不含参数

getCookies获得所有cookie对象

getSession获得与请求关联的HttpSession对象

2.获得请求头

getHeader(string name)方法
getHeaders(String name)方法
getHeaderNames方法

3.HttpServletRequst常用方法

==String getParameter(“username”)==：返回指定名字参数的值，最常用

String[] getParameterValues(“hobby”)：返回指定名字参数的多个参数值。(复选框)

Enumeration getParameterNames()：返回所有参数名的 Enumeration 对象

Map<String,String[]> getParameterMap()：返回所有的参数和值所组成的Map对象

setCharacterEncoding(“UTF-8”):设置请求体的字符集，可以用来解决中文乱码

getContextPath():是jsp中获取路径的一种方式，返回当前页面所在的应用的名字如/dmsd-itoo-exam-log-web

getServerPort()，返回当前页面所在的服务器使用的端口，80；

4.HttpServletResponse常用方法

OutputStream getOutputStream()：获取字节输出流对象( 文件下载)。

PrintWriter getWriter(): 获取字符输出流对象

response.setContentType(“text/html;charset=UTF-8”)，设置输出的MIME类型(内容的类型)

response.setCharacterEncoding(“UTF-8”)，response.setContentType(“text/html;charset=utf-8”)，设置输出数据的编码方式

public void addCookie(): 写入cookie

public void sendError(int sc,[string msg]):发送一个错误状态码，如果有msg就也发送msg（错误信息）

public void sendRedirect(String location): 从定向到新的url
